MAP IT, DRIVE IT!

Project Goal

The goal of this project was to explore the concept of "dead reckoning". This project is not a comprehensive demonstration of the concept; it is reletively simple, but simplifying things is key in gaining a comprehensive understanding. This project could me developed futher to be applicable in the real world. For example there has been a push in companies like amazon, to create robots that follow paths to collect and deleiver things across large warehouses. This project serves as a base for modeling obsticals and walls that the robot would have to go through.

Reflection

Procedural Abstraction:
I used proceduaral abstraction by creating several helper functions. One of these helper functions was the make_square() function. This function made a filled in square in what ever color was called with the function. This was used to set up the obsticals so it was easier to see the path. Using this function made the code more robust because I did not need to repeat the same lines of code over and over. I also made helper functions for each individual paths driving instructions. This way I can call the specific functions in one larger main function after the set up grid has been executed.

How Could This Be More Efficient?:
This code is not exactly efficient as it is all hard coded, and therefor is only partially reusable. In order to make this code more efficient I would use sensors so the robot knows when it is in contact with an obstical. This way the only information it would need is any 2D list.
